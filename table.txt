================================================================================
                              APP SCOPE
================================================================================

PROJECT: SRIMCA AI Assistant (Flutter Mobile App + Flask Backend)
PURPOSE: A college management system with AI-powered assistant for
         SRIMCA (Shrimad Rajchandra Institute of Management & Computer Applications)

TARGET USERS:
1. Admin - College administration staff
2. Student - Enrolled students
3. Faculty - Teaching staff
4. Visitor - External visitors (prospective students, guests)

TECHNOLOGY STACK:
-----------------
Frontend: Flutter (Dart)
Backend: Flask (Python)
Database: MongoDB Atlas
Authentication: JWT + Firebase
AI: RAG (Retrieval-Augmented Generation) with ChromaDB
Hosting: Render.com

================================================================================
                     DATA PREPROCESSING FOR AI RESPONSE
================================================================================

DATA SOURCE: SRIMCA Official Website
------------------------------------
The AI assistant uses data scraped/preprocessed from SRIMCA's official website
to provide accurate responses about the college.

DATA PREPROCESSING STEPS:
------------------------

1. DATA COLLECTION (backend/srimca/data/sample_srimca_info.txt)
    Manual extraction from SRIMCA official website
    Information collected:
      College full name: "Shrimad Rajchandra Institute of Management and Computer Application"
     * Location: Maliba Campus, Gopal Vidyanagar, Bardoli, Surat, Gujarat
     * University: Constituent college of Uka Tarsadia University
     * Programs: MCA (started 2002), MBA (started 2004) approved by AICTE
     * Academic calendar with semester dates
     * Daily timetable for MCA 2nd semester
     * Infrastructure details (computers, Wi-Fi, internet, library, auditorium)

2. DATA LOADING (backend/srimca/loader.py)
   - load_content(): Loads all .txt files from data directory
   - load_lines(): Loads text files as individual lines
   - Combines all text files into single content

3. TEXT CHUNKING (backend/srimca/build_db.py)
   - Split content by newlines into individual chunks
   - Each line becomes a separate knowledge item
   - Total chunks: ~30+ text segments

4. EMBEDDING GENERATION (backend/srimca/config.py)
   - Model: SentenceTransformer("all-MiniLM-L6-v2")
   - Type: FREE local embedding model (no API cost)
   - Creates 384-dimensional vectors for each text chunk

5. VECTOR STORAGE (backend/srimca/build_db.py)
   - MongoDB collection: "knowledge"
   - Stores each chunk with its embedding
   - Fields: "text", "embedding"
   - Vector search enabled for semantic matching

6. RAG PIPELINE (backend/srimca/retriever.py)
   - Semantic search using cosine similarity
   - Retrieves top-k most relevant chunks
   - Context window for GPT generation

7. ANSWER GENERATION (backend/srimca/gpt_answer.py)
   - Uses OpenAI GPT API (optional - requires API key)
   - Fallback to template-based responses if no API key
   - Combines retrieved context + user question

DATA FLOW FOR AI RESPONSE:
-------------------------
User Question → API Request → Embed Question → Vector Search (MongoDB) 
→ Retrieve Top-K Chunks → Generate Answer (GPT/Fallback) → Return Response
